# Демо / смок — Итерация 03 (LiteLLM access-layer: cost/latency SLO)

**Простыми словами, что тут построено.** У каждого LLM-вызова конвейера появился ценник и
секундомер: разобрал тикет — в журнале `logs/llm_calls.jsonl` осталась строка «какой тир, какая
модель, сколько миллисекунд, сколько долларов, уложился ли в бюджет». Бюджеты (максимум долларов и
миллисекунд на вызов) задаются переменными окружения, превышение громко помечается. А правила
обращения с LiteLLM («только SDK, никакого Proxy, телеметрия выключена») перестали быть текстом в
доке — их проверяет pytest.

**Что мы доказываем этим прогоном.** Три вещи, каждая — глазами скептика. Первое: след действительно
пишется, и цена в нём — настоящая (записана живым вызовом в кассету, реплей показывает её за $0).
Второе: бюджеты действительно срабатывают — ужимаем порог и видим BREACH-предупреждение. Третье:
security-гейт действительно проверяет дисциплину механически, а не на честном слове. Вместе это
north-star строка «Cost & latency SLO / бюджеты для LLM (LLM FinOps)» плюс живое переключение
`cheap`↔`smart` одной env-переменной (productionized routing).

Все команды копипаст-исполнимы из корня репозитория `/Users/dd/projects/pet/triagewise-lite`;
все шаги, кроме помеченного «⚠️ live», ничего не стоят (реплей по кассетам). Для шагов 3–6 нужен
поднятый MLflow (промпт триажа живёт в реестре с iter 1) — шаг 2 это обеспечивает.

Перед началом:
```bash
cd /Users/dd/projects/pet/triagewise-lite
```

## 2. Поднять реестр и убедиться, что промпт на месте

**Простыми словами:** CLI-триаж берёт промпт из MLflow-реестра (iter 1), поэтому реестр должен быть
поднят. Регистрация идемпотентна — повторный прогон не плодит версий.

(на свежем сторе — регистрирует v1/v2), без создания лишних версий при повторе.

## 3. Разобрать тикет дёшево — и увидеть след с настоящей ценой

**Простыми словами:** это главный артефакт итерации. Гоняем обычный триаж в replay ($0): на stderr
печатается SLO-строка, а в журнал дописывается JSON-запись. Цена в ней — не выдумка: это цена,
записанная живым nano-вызовом в кассету при record-прогоне.

```bash
uv run python -m app.cli.main DW-001
```
**Ожидаемо:** первая строка вывода — SLO-след (цена настоящая, режим replay, бюджет не нарушен), ниже
обычный JSON триажа:
```
[slo] cheap->gpt-4.1-nano-2025-04-14 (replay) 0ms $0.000038 (cassette) ok
[DW-001] cheap (replay)
{ "category": ..., "priority": "high", ... }
```

## 4. Переключить весь конвейер на умную модель — одной переменной

**Простыми словами:** это «productionized routing» вживую. Никаких правок кода: `TRIAGE_TIER=smart` —
и тот же тикет идёт через gpt-4.1 (по своей кассете, всё ещё $0). Заодно видно, что smart-вызов стоил
в ~25 раз дороже nano — ради этого сравнения журнал и заводили.

```bash
TRIAGE_TIER=smart uv run python -m app.cli.main DW-001
```
**Ожидаемо:**
```
[slo] smart->gpt-4.1-2025-04-14 (replay) 0ms $0.000962 (cassette) ok
[DW-001] smart (replay)
```

## 5. Прочитать журнал как данные — jq по JSONL

**Простыми словами:** след — это не строчки в терминале, а машиночитаемый журнал: одна JSON-запись
на вызов. Смотрим последние две записи (cheap и smart из шагов 3–4) в сжатом виде.

```bash
tail -2 logs/llm_calls.jsonl | jq -c '{ts, tier, model, mode, latency_ms, cost_usd, cost_source, slo_breaches}'
```
**Ожидаемо:** две записи с полным набором полей (jq печатает цену полным десятичным числом,
как она лежит в файле) —
```json
{"ts":"…","tier":"cheap","model":"gpt-4.1-nano-2025-04-14","mode":"replay","latency_ms":0.1,"cost_usd":0.000037699999999999995,"cost_source":"cassette","slo_breaches":[]}
{"ts":"…","tier":"smart","model":"gpt-4.1-2025-04-14","mode":"replay","latency_ms":0.4,"cost_usd":0.000962,"cost_source":"cassette","slo_breaches":[]}
```

## 6. Ужать бюджет — и увидеть, как SLO краснеет

**Простыми словами:** бюджет, который не умеет срабатывать, — не бюджет. Ставим порог цены ниже
стоимости smart-вызова ($0.000962) и убеждаемся, что access-layer предупреждает: BREACH в stderr и
именованное нарушение в журнале. Вызов при этом не валится — это наблюдаемость, не рубильник.

```bash
SLO_MAX_COST_USD=0.0005 TRIAGE_TIER=smart uv run python -m app.cli.main DW-001
tail -1 logs/llm_calls.jsonl | jq -c '{tier, cost_usd, slo_breaches}'
```
**Ожидаемо:** в stderr — `BREACH`, в журнале — названное нарушение:
```
[slo] smart->gpt-4.1-2025-04-14 (replay) 0ms $0.000962 (cassette) BREACH: cost $0.000962 > $0.000500
{"tier":"smart","cost_usd":0.000962,"slo_breaches":["cost $0.000962 > $0.000500"]}
```

## 7. Заглянуть в сам артефакт — кассета несёт цену и расход токенов

**Простыми словами:** показываем содержимое хранилища, а не только его отражение в логе. Каждая
кассета после record-прогона несёт не только ответ модели, но и `usage` (расход токенов) с
`cost_usd` — провенанс цены: видно, из чего она сложилась.

```bash
for f in cassettes/*.json; do jq -c '{model, cost_usd: .response.cost_usd, total_tokens: .response.usage.total_tokens}' "$f"; done
```
**Ожидаемо:** обе кассеты с реальными ценами записанных вызовов —
```json
{"model":"gpt-4.1-nano-2025-04-14","cost_usd":0.000037699999999999995,"total_tokens":182}
{"model":"gpt-4.1-2025-04-14","cost_usd":0.000962,"total_tokens":208}
```

callbacks пусты и телеметрия выключена до первого вызова, единственное место импорта, ленивость в
replay, версия запиннена) проверяются тестами по исходникам, `sys.modules` и lock-файлу. Нарушь любое —
тест упадёт.

**Ожидаемо:** пять тестов, все зелёные (список имён: no_proxy-поверхность, единственный import-site,
обнуление каналов до вызова, ленивость в replay, пин версии) —

## 9. ⚠️ live, стоит денег — перезапись кассет живыми вызовами

**Простыми словами:** так кассеты и получили настоящие цены. Повторять при каждом смоке не нужно —
только если изменился промпт/модель и кассеты надо перезаписать. Два вызова (nano + gpt-4.1) стоят
суммарно ~$0.001. Ключ должен лежать в `.env`: `echo 'OPENAI_API_KEY=sk-…' >> .env` (единственное
значение, которое знаете только вы).

```bash
LLM_MODE=record uv run python -m app.cli.main DW-001
LLM_MODE=record TRIAGE_TIER=smart uv run python -m app.cli.main DW-001
```
**Ожидаемо:** обе команды печатают SLO-строку с `mode=record`, `cost_source=live` и ненулевой ценой;
в `cassettes/` лежат две кассеты с `content`, `usage` и `cost_usd`; после этого шаги 3–7
воспроизводятся офлайн.
