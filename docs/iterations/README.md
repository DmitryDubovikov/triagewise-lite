# Журнал итераций

Каждая итерация хребта (см. `ROADMAP.md`) оставляет здесь подпапку `NN/` (двузначный номер)
с тремя документами, написанными для будущего-себя живой прозой, а не телеграфно:

- **`explainer.md`** — что итерация делает и зачем; по каждому новому инструменту — что это и
  как минимально подключили. Открывается баннером 🎯, затем «Зачем это (продукт и ценность)»,
  «🧵 Что это дало резюме», «TL;DR», схема потока данных и карта «где в коде».
- **`demo.md`** — happy-path смок-тест: пронумерованные копипаст-исполнимые шаги с ожидаемым
  результатом под каждым. Live-шаги (стоят денег) помечены явно.
- **`learnings.md`** — принятые решения и почему, грабли, отложенные TODO и осознанные срезы.

Существование инструмента — ворота итерации (existence-gate), а не точность триажа
(см. `CLAUDE.md` правило 1).
